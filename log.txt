La signature est valide maintenant, donc le crash vient du runtime. Pour diagnostiquer précisément, j'ai besoin du crash report.

Après un crash, sur ton Mac :


ls -t ~/Library/Logs/DiagnosticReports/ | grep -iE "code|ergopti" | head -3
Puis envoie-moi le début du plus récent :


cat ~/Library/Logs/DiagnosticReports/<filename>.ips | head -80
Surtout les champs exception, termination, Code Signing, et le premier thread — ça dira exactement pourquoi le kernel tue le process (signature mismatch / library validation / hardened-runtime violation / entitlement refus / quarantine).

En parallèle, essaie aussi :


xattr -cr ~/Applications/Ergopti.app
open ~/Applications/Ergopti.app
Si après xattr -cr l'app se lance correctement, c'est juste le flag com.apple.quarantine hérité du bundle source qu'il faut que j'ajoute dans le script.
